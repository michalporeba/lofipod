## Project Overview

`lofipod` is a TypeScript library for applications that keep working locally
first and synchronize canonical RDF data to a user's SOLID Pod in the
background. The repository is a single package with three public entrypoints:
the environment-neutral core package, a browser adapter entrypoint, and a Node
adapter entrypoint.

## Quick Reference

- **Tech Stack:** TypeScript, Node 24+, `n3`, `better-sqlite3`, Vitest, tsup
- **Primary Entry Point:** `src/index.ts`
- **Environment Entry Points:** `src/browser.ts`, `src/node.ts`
- **Architecture Pattern:** local transaction log plus materialized read model
- **Remote Model:** canonical Turtle resources plus app-private N-Triples log
- **Deployment:** npm package published from GitHub release workflow

## Getting Started

### Prerequisites

- Node.js `24+`
- npm
- Docker, only for `npm run test:pod`

### Setup

```bash
npm ci
```

### Run Locally

```bash
npm run demo -- help
```

### Run Tests

```bash
npm run verify
npm run build
npm run test:demo
npm run test:pod
```
